# Templates — comentário por ferramenta

## Jira / Linear / Azure DevOps

Plain text ou markdown leve:

```
*Resumo:* [uma linha]
*O que foi feito:* [2–3 frases]
*Evidência:* [como validou]
*Branch sugerida:* fix/1234-titulo-curto
```

## Trello / ClickUp

Card comment — tom direto:

```
Concluído: [título curto]

Antes: [problema em 1 frase]
Depois: [solução em 1–2 frases]
Validação: [teste/tela]
```

## Versão ultra-curta (standup / gestão)

```
✅ [Título] — [o que mudou para o usuário]. Validado: [evidência breve].
```

## Checklist antes de publicar

- [ ] Título claro
- [ ] Problema + solução distinguíveis
- [ ] Evidência real (não inventada)
- [ ] Linguagem acessível para gestão
- [ ] Branch sugerida (se aplicável)
- [ ] Ticket número referenciado
