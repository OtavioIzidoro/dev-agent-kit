# Frontend

- Reutilize componentes existentes antes de criar novos.
- Trate estados de loading, empty e erro em telas com dados assíncronos.
- Para telas novas com foco visual, use `/frontend-design`.
- Para replicar um print com fidelidade, use `/replicacao-fiel-tela-por-print`.
- Antes de criar componentes novos, use `/guardiao-padroes` para seguir padrões existentes.
- Para implementar tela, form, modal ou integração API, use `/front-implementador`.
